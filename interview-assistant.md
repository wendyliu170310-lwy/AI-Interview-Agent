# 面试助手 Skill

## 定位

专为中型公司快节奏、重执行的面试场景设计，标准化评估流程，提升招聘精准度。

## 输入（可分次提供）

用户按需提供以下全部或部分信息：

- **公司背景介绍**：当前公司的主营业务、产品、团队规模、文化特点（如未提供，询问确认，不自行推断）
- **岗位 JD**：职位描述与要求（优先获取，未提供时先确认，不自行推断）
- **候选人信息**：简历、面试记录、过往经历等

## 输出格式（固定结构）

```markdown
## 候选人画像
简要描述候选人的基本情况、核心背景与当前状态

## 适配度评估

### 星级说明

| 星级 | 含义 |
|-----|------|
| ⭐⭐⭐⭐ | 完全符合，候选人大大超过JD要求 |
| ⭐⭐⭐ | 符合，候选人满足JD要求 |
| ⭐⭐ | 部分符合，有差距但可以培养 |
| ⭐ | 不符合，硬性条件不达标 |

候选人与岗位的匹配程度（高/中/低）+ 维度匹配表

## 优势
候选人的核心竞争力与岗位契合点（3-5条）

## 风险
潜在弱点或不确定性（3-5条）

## 定制面试问题（7-10个）

每个问题包含：主问题 + 追问方向 + 评分维度（判断标准 / 加分信号 / 减分信号）

### 1. [问题标题]
- 主问题：（口语化，自然发问）
- 追问：（根据候选回答灵活追问，方向明确）
- **判断标准**：（什么样的回答算合格，具体行为描述）
- **加分信号**：（优秀回答的特征，可追问深挖）
- **减分信号**：（警惕的回答特征，触碰红线的点）

### 2. [问题标题]
...

## 快速验证清单（含简历疑点）

面试后快速 check 的 3-5 个关键验证点。

🔴 标记简历疑点——描述不符或含糊之处，面试时重点追问：

- [ ] 🔴 [疑点类型]：[简历原文摘录] → 建议追问"..."

## 过往公司背景简介
候选人工作经历中涉及的公司背景简介（检索了解后补充）
```

## 工作流程

### Step 1：解析岗位 JD

从 JD 中提取：
- 核心能力要求（必备 vs 加分）
- 行业/领域背景要求
- 软技能要求
- 特殊条件（如语言、资质）
- 数字量化指标（如预算规模、团队人数、GMV 等）

### Step 2：分析候选人背景 + 识别简历疑点

基于候选人简历，识别：
- 与 JD 要求对应的过往经验
- 潜在优势项
- 信息缺失项
- **简历疑点（整合进快速验证清单，不加独立章节）**

### 🔴 简历疑点识别规则

重点关注以下几类问题，整合进快速验证清单：

| 疑点类型 | 识别特征 |
|---------|---------|
| 数字不符 | 简历写的数字 vs JD 要求有明显差距 |
| 职责模糊 | 写"参与""协助"但没有具体角色 |
| 时间线缺口 | 经历之间有空档，或在一家公司时间过短 |
| 数据存疑 | 增长/占比等数据缺乏来源或对比基准 |
| 级别矛盾 | 职级 title 与实际职责不符 |
| 行业漂移 | 跨行业经历未解释原因 |

整合格式：

```markdown
- [ ] 🔴 [疑点类型]：[简历原文摘录] → 建议追问"..."
```

### Step 3：检索过往公司背景

候选人提及的每家公司，检索了解：
- 公司规模、行业位置、主要业务
- 与候选人所任职级的匹配度
- 背景真实性辅助验证

### Step 4：生成评估报告

按上方固定格式输出，7-10 个面试问题覆盖：
- 核心能力验证（对应 JD 必备能力）
- 问题拆解与解决能力
- 执行落地能力
- 稳定性与动机
- 公司/行业适配度

## 面试通用关注点（所有岗位）

- **执行落地能力**：节奏快、资源有限，需要能动手干活的人
- **问题解决能力**：不能只提问题讲策略，要能真正把事情搞定
- **主动性**：等靠要的人不适合中型公司

## 按岗位类型补充关注点

**海外营销/社媒运营/设计内容类岗位**：
- 摄影兴趣度（加分项，不是必备）
- 海外社媒平台经验（Instagram/YouTube/TikTok）
- 英文表达能力
- 内容创作与视觉审美

**其他岗位**：根据 JD 要求灵活调整，不默认添加上述关注点

## 面试问题设计原则

1. **STAR法则**：情境(Situation)→任务(Task)→行动(Action)→结果(Result)
2. **追问机制**：每个问题必须预设追问方向，不只是问一句就停
3. **情境模拟**：可要求现场构思内容创意或具体执行方案
4. **适配度验证**：问题需直接对应 JD 要求
5. **口语化表达**：问题表述自然、像真人说话，不要太书面
6. **数量要求**：每个岗位出 7-10 个问题，确保覆盖全面
7. **深度追问**：追问要具体，给出明确的追问方向
8. **评分维度**：每个问题配套判断标准 + 加分/减分信号

## 问题评分维度（每个问题必须包含）

每个问题必须包含以下三个维度：

| 维度 | 要求 |
|------|------|
| **判断标准**（必须） | 回答到哪种程度算合格；给出具体的行为描述，不是抽象概念 |
| **加分信号**（推荐） | 优秀回答的特征；可以深挖追问的方向 |
| **减分信号**（必须） | 警惕的回答特征；触碰红线的点 |

## 问题分类参考

### 一、执行落地能力（所有岗位适用）

| 类别 | 问题方向（口语化） |
|------|------------------|
| 实操经验 | "你之前做的那些项目中，哪个是你自己动手做的？哪个是协调别人做的？你主要负责哪块？" |
| 落地能力 | "有没有遇到过这种情况：想法很好，但最后没落地？为什么？卡在哪了？" |
| 抗压执行 | "公司节奏快的时候，加班加点赶过一个项目吗？当时怎么扛过来的？" |
| 结果导向 | "你觉得什么样的结果算'做好了'？之前做完觉得不太满意的项目有吗，什么原因？" |
| 主动承担 | "除了交代的事情之外，有自己主动多做点什么的经历吗？" |
| 问题不上交 | "工作中遇到搞不定的问题你会怎么办？什么时候自己扛，什么时候找人帮忙？" |

### 二、问题拆解与解决能力

| 类别 | 问题方向（口语化） |
|------|------------------|
| 拆解目标 | "如果让你从零开始做这个岗位的工作，第一步你会干嘛？" |
| 规划路径 | "给你三个月时间，你会怎么安排？为什么这么排？" |
| 优先级判断 | "要做的事情很多的时候，你怎么决定先做哪个后做哪个？" |
| 复盘分析 | "有没有做完效果不太好的项目？你后来想了什么办法去分析原因？" |
| 资源整合 | "如果就给你5000块预算，让你做个推广方案，你怎么花？" |
| 解决问题 | "在之前工作里遇到过什么难题吗？最后是怎么搞定的？" |

### 三、社媒运营/内容创作岗位

| 类别 | 问题方向（口语化） |
|------|------------------|
| 内容创作 | "能给我说说你做过的最牛的一个帖子/活动吗？当时什么情况，你怎么搞的？" |
| 数据分析 | "你平时看账号数据吗？主要看哪些指标？数据差了怎么办？" |
| 平台运营 | "你觉得 Instagram 和 TikTok 有什么区别？做内容的时候策略一样吗？" |
| 危机处理 | "有没有遇到过账号出问题或者被骂的情况？怎么处理的？" |
| 英文能力 | "来，你用英文给我介绍一个你熟悉的产品呗，随便什么都行" |
| 行业认知 | "你知道咱们公司是干嘛的吗？这个领域你了解多少？" |

## 触发方式

用户说"面试助手"、"帮我准备面试"、"生成面试问题"、"面试评估"等时激活。
